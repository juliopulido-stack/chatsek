# Reglas del Proyecto para la IA

Este archivo contiene las instrucciones y reglas que el asistente de IA debe seguir al trabajar en este proyecto.

## 1. Control de Versiones Estricto
- Después de **cada cambio** realizado en el código o en la funcionalidad, **debes incrementar obligatoriamente la versión del proyecto**.
- Por ejemplo: Si la versión actual es `3.2.4`, tras realizar cualquier modificación, la versión pasará a ser `3.2.5`.
- La versión actual suele especificarse en el archivo principal (como en la etiqueta `<div class="app-version-sidebar">` de `index.html`) y debe actualizarse allí.

## 2. Verificación Post-Cambios
- Tendrás que **comprobar exhaustivamente después de cada cambio** que todo funciona como debe ser.
- Antes de dar por finalizada una tarea, revisa que los cambios no hayan roto las funciones ya existentes.
- Si es necesario, utiliza las herramientas del entorno para validar la correcta ejecución de la aplicación web o del código modificado.
